# Codex 웹

원본 URL: https://developers.openai.com/codex/cloud

Codex는 코드를 읽고, 수정하고, 실행할 수 있는 OpenAI의 코딩 에이전트입니다. 더 빠르게 빌드하고, 버그를 수정하며, 낯선 코드를 이해하도록 도와줍니다. Codex cloud를 사용하면 Codex가 자체 클라우드 환경에서 (병렬 실행을 포함해) 백그라운드 작업을 수행할 수 있습니다.

## Codex 웹 설정

[Codex](https://chatgpt.com/codex)로 이동해 GitHub 계정을 연결하세요. 그러면 Codex가 저장소의 코드를 다루고, 작업 결과로 Pull Request를 만들 수 있습니다.

Plus, Pro, Business, Edu, Enterprise 요금제에는 Codex가 포함되어 있습니다. 포함 항목은 [what’s included](https://developers.openai.com/codex/pricing)에서 자세히 확인하세요. 일부 Enterprise 워크스페이스는 Codex에 접근하기 전에 [admin setup](https://developers.openai.com/codex/enterprise/admin-setup)이 필요할 수 있습니다.

## Codex 웹 활용

### [프롬프트 작성법 알아보기 더 명확한 프롬프트를 작성하고, 제약을 추가하며, 더 나은 결과를 위해 적절한 세부 수준을 선택하세요.](https://developers.openai.com/codex/prompting#prompts)
### [공통 워크플로 알아보기 검증된 패턴으로 작업을 위임하고, 변경 사항을 검토하며, 결과를 PR로 전환하세요.](https://developers.openai.com/codex/workflows)
### [환경 구성 Codex가 클라우드에서 작업을 실행할 때 사용할 리포지터리, 설정 단계, 도구를 선택하세요.](https://developers.openai.com/codex/cloud/environments)
### [IDE 확장에서 작업 위임 편집기에서 클라우드 작업을 시작하고 진행 상황을 모니터링한 뒤, 생성된 diff를 로컬에 적용하세요.](https://developers.openai.com/codex/ide/features#cloud-delegation)
### [GitHub에서 위임하기 이슈와 Pull Request에 `@codex`를 태그해 작업을 생성하고 GitHub에서 바로 변경 사항을 제안하세요.](https://developers.openai.com/codex/integrations/github)
### [인터넷 접속 제어 Codex가 클라우드 환경에서 공용 인터넷에 접근할 수 있을지, 그리고 언제 활성화할지를 결정하세요.](https://developers.openai.com/codex/cloud/internet-access)
