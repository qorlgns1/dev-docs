# 로컬 환경

소스 URL: https://developers.openai.com/codex/app/local-environments

로컬 환경을 사용하면 워크트리의 설정 단계와 프로젝트의 공통 작업을 구성할 수 있습니다.

로컬 환경은 [Codex 앱 설정](codex://settings) 패널에서 구성합니다. 생성된 파일을 프로젝트의 Git 리포지토리에 체크인해 다른 사람과 공유할 수 있습니다.

Codex는 이 구성을 프로젝트 루트의 `.codex` 폴더에 저장합니다. 리포지토리에 프로젝트가 둘 이상 있다면, 공유 `.codex` 폴더가 포함된 프로젝트 디렉터리를 여세요.

## 설정 스크립트

워크트리는 로컬 작업과 다른 디렉터리에서 실행되므로, 프로젝트 설정이 완전하지 않거나 리포지토리에 체크인되지 않은 의존성 또는 파일이 누락되어 있을 수 있습니다. 설정 스크립트는 새 스레드 시작 시 Codex가 새 워크트리를 만들 때 자동으로 실행됩니다.

의존성 설치나 빌드 프로세스 실행처럼, 환경 구성에 필요한 명령을 실행하려면 이 스크립트를 사용하세요.

예를 들어 TypeScript 프로젝트에서는 설정 스크립트로 의존성을 설치하고 초기 빌드를 수행할 수 있습니다:
[code] 
    npm install
    npm run build
[/code]

설정이 플랫폼별로 다르다면 macOS, Windows, Linux용 설정 스크립트를 정의해 기본값을 재정의하세요.

## 액션

액션을 사용해 앱 개발 서버 시작이나 테스트 스위트 실행 같은 공통 작업을 정의하세요. 이 액션들은 빠르게 접근할 수 있도록 Codex 앱 상단 바에 표시됩니다. 액션은 앱의 [통합 터미널](https://developers.openai.com/codex/app/features#integrated-terminal) 내에서 실행됩니다.

액션은 프로젝트 빌드 트리거나 개발 서버 시작처럼 자주 사용하는 작업을 매번 입력하지 않도록 도와줍니다. 일회성 빠른 디버깅에는 통합 터미널을 직접 사용할 수 있습니다.

예를 들어 Node.js 프로젝트에서는 다음 스크립트를 포함하는 “Run” 액션을 만들 수 있습니다:
[code] 
    npm start
[/code]

액션의 명령이 플랫폼별로 다르다면 macOS, Windows, Linux용 플랫폼별 스크립트를 정의하세요.

액션을 식별하기 위해 각 액션에 연결된 아이콘을 선택하세요.
