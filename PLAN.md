# dev-docs 개선 계획서

> 목표: 개발자 문서 한국어 번역 사이트 → 광고 수익이 발생하는 지속 가능한 서비스

---

## 현재 상태

- **사이트**: https://dev-docs.moodybeard.com
- **콘텐츠**: Codex (62p), Next.js (411p) — 한국어 + 영어
- **파이프라인**: `$add-doc <url>` 스킬로 URL → 번역 → 배포 자동화
- **인프라**: Astro Starlight + Vercel

---

## Phase 1 — 기반 강화 (1~2개월)

### 1-1. 콘텐츠 확장

광고 수익은 **트래픽 규모**에 비례합니다. 페이지 수를 늘리는 것이 최우선입니다.

| 우선순위 | 문서 소스 | 예상 페이지 수 |
|---------|----------|-------------|
| ★★★ | React 공식 문서 | ~50p |
| ★★★ | TypeScript Handbook | ~60p |
| ★★☆ | Node.js 문서 | ~100p |
| ★★☆ | Tailwind CSS | ~80p |
| ★☆☆ | Astro 문서 | ~60p |

→ `$add-doc` 스킬로 추가 가능. 목표: **1,000페이지 이상**

### 1-2. SEO 강화

현재 기본적인 SEO만 적용된 상태. 검색 유입을 늘리려면 추가 작업 필요.

- [ ] **`sitemap.xml` Google Search Console 등록** — 크롤링 가속
- [ ] **`robots.txt` 최적화** — 불필요한 경로 차단
- [ ] **Canonical URL 설정** — en/ko 중복 콘텐츠 처리
  ```js
  // astro.config.mjs
  head: [{ tag: 'link', attrs: { rel: 'canonical', href: '...' } }]
  ```
- [ ] **hreflang 태그** — 언어별 페이지 연결 (Google 국제화 SEO)
  ```html
  <link rel="alternate" hreflang="ko" href="https://dev-docs.moodybeard.com/codex/" />
  <link rel="alternate" hreflang="en" href="https://dev-docs.moodybeard.com/en/codex/" />
  ```
- [ ] **Open Graph 이미지** — 소셜 공유 시 미리보기 이미지 생성

### 1-3. 필수 법적 페이지 (광고 승인 전제 조건)

Google AdSense 승인을 위해 **반드시 필요**한 페이지입니다.

- [ ] `/privacy` — 개인정보처리방침 (쿠키, 광고 데이터 수집 명시)
- [ ] `/about` — 사이트 소개, 운영자 정보
- [ ] `/contact` — 연락처 (이메일 또는 폼)

## Phase 2 — 트래픽 확보 (2~4개월)

광고 수익은 **월 1만 PV 이상**부터 의미 있는 수준이 됩니다.

### 2-1. 검색 트래픽 분석

- [ ] **Google Analytics 4 연동** — 유입 경로, 인기 페이지 파악
  ```js
  // astro.config.mjs head에 GA 스크립트 추가
  ```
- [ ] **Google Search Console** — 검색 노출 키워드, 클릭률 모니터링
- [ ] 인기 페이지 기반으로 **번역 품질 우선 개선**

### 2-2. 번역 품질 개선

현재 자동 번역 품질이 고르지 않습니다.

- [ ] **style-guide 강화** — 자주 쓰이는 기술 용어 통일 (예: `build` → 빌드/빌드하다)
- [ ] **번역 후처리 스크립트** — 잘못된 패턴 자동 교정
  - `"마지막 업데이트"` 같은 불필요한 메타 텍스트 제거
  - 브레드크럼 링크 정리
- [ ] **고트래픽 페이지 수동 검토** — 상위 20페이지 품질 보장

### 2-3. 사이트 UX 개선

- [ ] **홈페이지 개선** — 섹션 카드, 최근 추가된 문서, 검색창
- [ ] **404 페이지 커스터마이징** — 관련 문서 추천
- [ ] **다크모드** — Starlight 기본 지원, 설정만 추가
- [ ] **번역 상태 표시** — 아직 번역 안 된 페이지 뱃지

## Phase 3 — 광고 수익화 (4~6개월)

### 3-1. Google AdSense 신청 조건 체크

| 조건 | 현재 상태 | 목표 |
|------|----------|------|
| 오리지널 콘텐츠 | ⚠️ 번역 콘텐츠 | 번역 출처 명시 + 편집 가치 추가 |
| 콘텐츠 양 | ✅ 470p+ | 1,000p+ 목표 |
| 사이트 나이 | 신규 | 3개월+ 운영 |
| 개인정보처리방침 | ❌ 없음 | Phase 1에서 추가 |
| 트래픽 | 측정 필요 | 일 500+ PV |

> ⚠️ 번역 사이트는 AdSense 심사가 까다롭습니다. **편집 가치**(요약, 비교, 튜토리얼 등)를 추가하면 승인 가능성이 높아집니다.

### 3-2. 광고 배치 전략

Starlight 커스텀 컴포넌트로 광고 슬롯 추가:

```
[사이드바 하단]     — 300x250 디스플레이 광고
[본문 상단]         — 728x90 리더보드 (데스크톱)
[본문 중간]         — 인피드 광고 (긴 문서)
[모바일 하단 고정]  — 320x50 앵커 광고
```

- Starlight `<Aside>` 슬롯 활용
- 콘텐츠 방해 최소화 — 독자 이탈률 관리

### 3-3. AdSense 대안

AdSense 승인이 어려울 경우:

| 대안 | 특징 |
|------|------|
| **Carbon Ads** | 개발자 타겟, 승인 쉬움, 단가 높음 |
| **EthicalAds** | 개발자 특화, 프라이버시 친화적 |
| **Kakao AdFit** | 한국어 사이트에 유리 |

## Phase 4 — 장기 성장 (6개월+)

### 4-1. 콘텐츠 차별화

단순 번역을 넘어 **편집 가치** 추가:
- 각 섹션마다 **"한국어로 보는 요약"** 페이지 추가
- **버전별 변경사항** 페이지 (Next.js 업데이트 추적)
- **비교 가이드** (예: App Router vs Pages Router)

### 4-2. 자동화 고도화

- [ ] **GitHub Actions 스케줄러** — 원본 문서 변경 감지 후 자동 번역/배포
  ```yaml
  on:
    schedule:
      - cron: '0 3 * * 1'  # 매주 월요일 새벽 3시
  ```
- [ ] **번역 diff 감지** — 원본 업데이트 시 해당 페이지만 재번역
- [ ] **Slack/Discord 알림** — 새 문서 추가 시 알림

### 4-3. 커뮤니티 & SEO

- [ ] RSS 피드 (`/rss.xml`) — 구독자 확보
- [ ] **번역 기여 가이드** — 오역 제보, PR 받기
- [ ] **소셜 미디어** — X(트위터), 오픈카톡 개발자 커뮤니티 공유

```

└── 콘텐츠 확장 (React, TypeScript 등)

1~2개월 안에
├── Google Analytics 연동
├── hreflang / canonical 태그
└── 번역 품질 개선 (고트래픽 페이지)

3~4개월 안에
├── AdSense 또는 Carbon Ads 신청
└── 광고 슬롯 컴포넌트 개발

6개월+
├── GitHub Actions 자동화
└── 커뮤니티 채널 운영
```
