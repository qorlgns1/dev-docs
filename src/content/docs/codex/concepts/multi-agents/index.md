# 멀티 에이전트

Source URL: https://developers.openai.com/codex/concepts/multi-agents

Codex는 전문화된 에이전트를 병렬로 생성해 실행하고, 그 결과를 하나의 응답으로 수집하여 멀티 에이전트 워크플로를 수행할 수 있습니다.

이 페이지는 핵심 개념과 트레이드오프를 설명합니다. 설정, 에이전트 구성, 예시는 [Multi-agents](https://developers.openai.com/codex/multi-agent)를 참고하세요.

## 멀티 에이전트 워크플로가 도움 되는 이유

컨텍스트 창이 커도 모델에는 한계가 있습니다. 요구 사항, 제약, 의사결정 등을 정의하는 메인 대화에 탐색 노트, 테스트 로그, 스택 트레이스, 명령 출력 같은 시끄러운 중간 결과를 쏟아 넣으면 세션 신뢰도가 시간이 지날수록 떨어질 수 있습니다.

이는 종종 다음과 같이 표현됩니다:

  * **컨텍스트 오염**: 유용한 정보가 시끄러운 중간 결과에 묻힙니다.
  * **컨텍스트 부패**: 덜 관련된 세부사항이 대화를 채우면서 성능이 저하됩니다.



배경 지식은 Chroma의 [context rot](https://research.trychroma.com/context-rot) 글을 참고하세요.

멀티 에이전트 워크플로는 시끄러운 작업을 메인 스레드 밖으로 옮겨 해결합니다:

  * **메인 에이전트**는 요구 사항, 의사결정, 최종 결과에 집중하게 합니다.
  * 탐색, 테스트, 로그 분석을 위해 전문화된 **서브 에이전트**를 병렬로 실행합니다.
  * 원시 중간 출력 대신 서브 에이전트의 **요약**을 반환합니다.



시작점으로는 대부분 읽기 위주인 작업(탐색, 테스트, 분류, 요약)에 병렬 에이전트를 사용하세요. 여러 에이전트가 동시에 코드를 수정하는 병렬 쓰기 워크플로는 충돌과 조정 비용이 커질 수 있으니 더 신중하게 다뤄야 합니다.

## 핵심 용어

Codex는 멀티 에이전트 워크플로에서 다음과 같은 용어를 사용합니다:

  * **멀티 에이전트**: Codex가 여러 에이전트를 병렬로 실행하고 결과를 결합하는 워크플로.
  * **서브 에이전트**: 특정 작업을 처리하도록 Codex가 시작하는 위임된 에이전트.
  * **에이전트 스레드**: 에이전트용 CLI 스레드로, `/agent` 명령으로 확인하거나 전환할 수 있습니다.



## 모델과 추론 선택

에이전트마다 필요한 모델과 추론 설정이 다릅니다.

`gpt-5.3-codex-spark`는 ChatGPT Pro 구독자를 위한 리서치 프리뷰로 제공됩니다. 최신 제공 상황은 [Models](https://developers.openai.com/codex/models)를 참고하세요. Codex API를 사용할 경우 현재는 GPT-5.2-Codex를 사용하세요.

### 모델 선택

  * **`gpt-5.3-codex`**: 코드 리뷰, 보안 분석, 다단계 구현, 모호한 요구사항처럼 강한 추론이 필요한 에이전트에 사용하세요. 메인 에이전트와 수정 제안·적용 에이전트가 여기에 해당합니다.
  * **`gpt-5.3-codex-spark`**: 속도가 중요하고 깊이가 덜 필요한 탐색, 읽기 많은 스캔, 빠른 요약 작업에 사용하세요. Spark는 요약된 결과를 메인 에이전트에 전달하는 병렬 워커에 적합합니다.



### 추론 강도 (`model_reasoning_effort`)

  * **`high`**: 복잡한 로직을 추적하거나 가정을 검증하고 엣지 케이스를 처리해야 할 때 사용(예: 리뷰어나 보안 중심 에이전트).
  * **`medium`**: 대부분의 에이전트에 적합한 균형 잡힌 기본값.
  * **`low`**: 작업이 단순하고 속도가 가장 중요할 때 사용.



추론 강도를 높이면 응답 시간과 토큰 사용량이 늘지만, 복잡한 작업의 품질은 향상될 수 있습니다. 자세한 내용은 [Models](https://developers.openai.com/codex/models), [Config basics](https://developers.openai.com/codex/config-basic), [Configuration Reference](https://developers.openai.com/codex/config-reference)를 참고하세요.
