# 거버넌스 및 가시성

Codex는 엔터프라이즈 팀에 도입 현황과 영향에 대한 가시성을 제공하고, 보안 및 규정 준수 프로그램을 위한 감사 가능성도 보장합니다. 일상적인 추적에는 셀프서비스 대시보드를, 프로그래매틱 보고에는 Analytics API를, 거버넌스 스택으로 상세 로그를 내보내려면 Compliance API를 사용하세요.

## Codex 사용량 추적 방법

필요에 따라 Codex 사용량을 모니터링하는 세 가지 방법이 있습니다:

  * **Analytics Dashboard**: 도입 상황과 코드 리뷰 영향에 빠르게 접근.
  * **Analytics API**: 구조화된 일별 지표를 데이터 웨어하우스나 BI 도구로 가져오기.
  * **Compliance API**: 감사·모니터링·조사를 위한 상세 활동 로그 내보내기.

## Analytics Dashboard

### 대시보드

[Analytics dashboard](https://chatgpt.com/codex/settings/analytics)는 ChatGPT 워크스페이스 관리자에게 기능 도입 현황을 추적할 수 있는 기능을 제공합니다.

Codex는 다음과 같은 대시보드를 제공합니다:

  * 제품별 일별 사용자 수(CLI, IDE, 클라우드, Code Review)
  * 일별 코드 리뷰 사용자 수
  * 일별 코드 리뷰 수
  * 우선순위별 코드 리뷰
  * 피드백 감성별 일별 코드 리뷰
  * 일별 클라우드 작업 수
  * 일별 클라우드 사용자 수
  * 일별 VS Code 확장 사용자 수
  * 일별 CLI 사용자 수

### 데이터 내보내기

관리자는 Codex 분석 데이터를 CSV 또는 JSON 형식으로 내보낼 수도 있습니다. 제공되는 내보내기 옵션은 다음과 같습니다:

  * 코드 리뷰 사용자 및 리뷰(Code Review에서 완료된 일별 고유 사용자와 총 리뷰 수)
  * 코드 리뷰 발견 사항 및 피드백(댓글, 반응, 답글, 우선순위별 발견 사항의 일별 수)
  * 클라우드 사용자 및 작업(일별 고유 클라우드 사용자와 완료된 작업 수)
  * CLI 및 VS Code 사용자(Codex CLI와 VS Code 확장용 일별 고유 사용자 수)
  * 사용자별 세션 및 메시지(Codex 사용자가 각 채널에서 시작한 일별 세션과 메시지 수)

## Analytics API

보고 자동화, 내부 대시보드 구축, 기존 엔지니어링 데이터와 Codex 지표 결합이 필요할 때 [Analytics API](https://chatgpt.com/codex/settings/apireference)를 사용하세요.

### 측정 내용

Analytics API는 워크스페이스의 일별 시계열 지표를 제공하며, 선택적으로 사용자별 분해와 클라이언트별 사용량도 지원합니다.

### 엔드포인트

#### 일별 사용량 및 도입

  * 스레드, 턴, 크레딧의 일별 총합
  * 클라이언트 인터페이스별 분해
  * 도입 및 파워 유저 분석을 위한 선택적 사용자별 보고

#### 코드 리뷰 활동

  * Codex가 완료한 풀 리퀘스트 리뷰
  * Codex가 생성한 총 댓글 수
  * 댓글 심각도 분해

#### 코드 리뷰에 대한 사용자 참여

  * Codex 댓글에 대한 답글
  * 업보트·다운보트를 포함한 반응
  * 팀이 Codex 피드백에 응답하는 방식에 대한 참여 분해

### 작동 방식

Analytics는 일별·시간 창 기반이며, 결과는 시간 순으로 정렬되고 커서 기반 페이지네이션으로 반환됩니다. 워크스페이스 단위로 쿼리하고, 선택적으로 사용자별 그룹화 또는 워크스페이스 수준 집계를 수행할 수 있습니다.

### 일반적인 사용 사례

  * 엔지니어링 관측 대시보드
  * 리더십 보고용 도입 현황
  * 사용량 거버넌스 및 비용 모니터링

## Compliance API

보안, 법무, 거버넌스 워크플로우를 위한 감사 가능한 기록이 필요할 때 [Compliance API](https://chatgpt.com/admin/api-reference)를 사용하세요.

### 측정 내용

Compliance API는 Codex 활동에 대한 로그와 메타데이터를 내보낼 수 있는 방법을 제공하여, 기존 감사·모니터링·보안 워크플로우에 데이터를 연결할 수 있게 합니다. eDiscovery, DLP, SIEM 같은 컴플라이언스 시스템과 함께 사용할 수 있도록 설계되었습니다.

### 내보낼 수 있는 항목

#### 활동 로그

  * Codex로 전송된 프롬프트 텍스트
  * Codex가 생성한 응답
  * 워크스페이스, 사용자, 타임스탬프, 모델과 같은 식별자
  * 토큰 사용량과 관련 요청 메타데이터

#### 감사 및 조사용 메타데이터

기록 메타데이터를 사용해 다음과 같은 질문에 답할 수 있습니다:

* 누가 작업을 실행했는지
  * 언제 실행했는지
  * 어떤 모델이 사용되었는지
  * 얼마나 많은 콘텐츠가 처리되었는지



#### 일반적인 사용 사례

  * 보안 조사
  * 규정 준수 보고
  * 정책 집행 감사
  * 이벤트를 SIEM 및 eDiscovery 파이프라인으로 라우팅



### 제공하지 않는 항목

  * 생성된 코드 라인 수(생산성의 시끄러운 대리 지표이며 잘못된 행동을 유도할 수 있음)
  * 제안 수용률(사용자가 보통 먼저 변경을 수락하므로 거의 100%)
  * 코드 품질 또는 성능 KPI



## 권장 패턴

대부분의 엔터프라이즈는 다음을 조합해 사용합니다:

  1. **Analytics Dashboard**: 셀프 서비스 모니터링과 빠른 응답
  2. **Analytics API**: 자동 보고 및 BI 통합
  3. **Compliance API**: 감사 내보내기와 조사
