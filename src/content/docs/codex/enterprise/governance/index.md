# 거버넌스 및 관찰 가능성

Codex는 엔터프라이즈 팀에 도입 및 영향력에 대한 가시성과 보안/규정 준수 프로그램에 필요한 감사 가능성을 제공합니다. 일상적인 추적에는 셀프 서브 대시보드를, 프로그램 보고에는 Analytics API를, 거버넌스 스택으로 상세 로그를 내보내려면 Compliance API를 사용하세요.

## Codex 사용량 추적 방법

필요에 따라 Codex 사용량을 모니터링하는 세 가지 방법이 있습니다:

- **Analytics Dashboard**: 도입 및 코드 리뷰 영향에 관해 빠르게 확인할 수 있습니다.
- **Analytics API**: 구조화된 일일 지표를 데이터 웨어하우스나 BI 도구로 가져옵니다.
- **Compliance API**: 감사, 모니터링, 조사 용도로 상세 활동 로그를 내보냅니다.

### 대시보드

[analytics dashboard](https://chatgpt.com/codex/settings/analytics)는 ChatGPT 워크스페이스 관리자가 기능 도입률을 추적할 수 있도록 합니다.

Codex는 다음 대시보드를 제공합니다:

- 제품별 일일 사용자 (CLI, IDE, 클라우드, 코드 리뷰)
- 일일 코드 리뷰 사용자
- 일일 코드 리뷰
- 우선순위 수준별 코드 리뷰
- 피드백 감성별 일일 코드 리뷰
- 일일 클라우드 작업
- 일일 클라우드 사용자
- 일일 VS Code 확장 사용자
- 일일 CLI 사용자

### 데이터 내보내기

관리자는 Codex 분석 데이터를 CSV 또는 JSON 형식으로 내보낼 수도 있습니다. Codex는 다음 내보내기 옵션을 제공합니다:

- 코드 리뷰 사용자 및 리뷰 (Code Review에서 완료된 일일 고유 사용자 및 총 리뷰 수)
- 코드 리뷰 결과 및 피드백 (댓글, 반응, 답글, 우선순위 수준 결과의 일일 집계)
- 클라우드 사용자 및 작업 (일일 고유 클라우드 사용자 및 완료된 작업)
- CLI 및 VS Code 사용자 (Codex CLI와 VS Code 확장에 대한 일일 고유 사용자)
- 사용자별 세션 및 메시지 (각 Codex 사용자의 각 표면에서 일일 세션 시작 및 메시지 수)

## Analytics API

자동화된 보고를 만들거나 내부 대시보드를 구축하거나 Codex 지표를 기존 엔지니어링 데이터와 결합하려면 [Analytics API](https://chatgpt.com/codex/settings/apireference)를 사용하세요.

### 측정 항목

Analytics API는 워크스페이스에 대한 일일 시계열 지표를 제공하며, 선택적으로 사용자별 세분화 및 클라이언트별 사용량을 포함할 수 있습니다.

### 엔드포인트

#### 일일 사용량 및 도입

- 스레드, 턴, 크레딧의 일일 합계
- 클라이언트 표면별 분류
- 도입 및 파워 유저 분석을 위한 선택적 사용자별 보고

#### 코드 리뷰 활동

- Codex가 완료한 풀 리퀘스트 리뷰
- Codex가 생성한 총 댓글
- 댓글의 심각도 분류

#### 코드 리뷰와 사용자 참여

- Codex 댓글에 대한 답글
- 찬성/반대 등 반응
- 팀이 Codex 피드백에 어떻게 대응하는지에 대한 참여 분포

### 작동 방식

Analytics는 일일 단위이며 시간 창이 지정됩니다. 결과는 시간순으로 정렬되며 커서 기반 페이지네이션으로 반환됩니다. 워크스페이스별로 쿼리하거나 선택적으로 사용자별로 그룹화하거나 워크스페이스 수준에서 집계할 수 있습니다.

### 일반적인 사용 사례

- 엔지니어링 관찰 가능성 대시보드
- 리더십 업데이트용 도입 보고
- 사용 거버넌스 및 비용 모니터링

## Compliance API

보안, 법무, 거버넌스 워크플로우에 감사 가능한 기록이 필요할 때는 [Compliance API](https://chatgpt.com/admin/api-reference)를 사용하세요.

### 측정 항목

Compliance API는 Codex 활동 로그와 메타데이터를 내보낼 수 있어 기존 감사, 모니터링, 보안 워크플로우에 연결할 수 있습니다. eDiscovery, DLP, SIEM 또는 기타 규정 준수 시스템과 함께 사용하도록 설계되었습니다.

### 내보낼 수 있는 항목

#### 활동 로그

- Codex에 전송된 프롬프트 텍스트
- Codex가 생성한 응답
- 워크스페이스, 사용자, 타임스탬프, 모델과 같은 식별자
- 토큰 사용량 및 관련 요청 메타데이터

#### 감사 및 조사용 메타데이터

기록 메타데이터를 사용하여 다음 질문에 답할 수 있습니다:

- 누가 작업을 실행했는가
- 언제 실행되었는가
- 어떤 모델이 사용되었는가
- 얼마나 많은 콘텐츠가 처리되었는가

#### 일반적인 사용 사례

- 보안 조사
- 규정 준수 보고
- 정책 시행 감사
- SIEM 및 eDiscovery 파이프라인으로의 이벤트 라우팅

### 제공하지 않는 항목

- 생성된 코드 라인 수 (생산성에 대한 다소 노이즈가 많은 대리 지표이고 잘못된 행동을 조장할 수 있음)
- 제안 수용률 (사용자가 일반적으로 먼저 변경을 수락하기 때문에 거의 100%)
- 코드 품질 또는 성능 KPI

## 추천 패턴

대부분의 엔터프라이즈는 다음 조합을 사용합니다:

1. 빠른 확인과 셀프 서브 모니터링을 위한 **Analytics Dashboard**
2. 자동화된 보고 및 BI 통합을 위한 **Analytics API**
3. 감사 내보내기 및 조사용 **Compliance API**
