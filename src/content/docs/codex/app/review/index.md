# 리뷰

원본 URL: https://developers.openai.com/codex/app/review

리뷰 패널은 Codex가 무엇을 변경했는지 이해하고, 대상이 명확한 피드백을 제공하며, 무엇을 유지할지 결정하는 데 도움이 됩니다.

이 기능은 Git 리포지토리 내부에 있는 프로젝트에서만 작동합니다. 프로젝트가 아직 Git 리포지토리가 아니라면, 리뷰 패널에서 리포지토리를 만들도록 안내합니다.

## 표시되는 변경 사항

리뷰 패널은 Codex가 편집한 내용만이 아니라 Git 리포지토리의 상태를 반영합니다. 즉, 다음이 표시됩니다.

  * Codex가 만든 변경 사항
  * 사용자가 직접 만든 변경 사항
  * 리포지토리의 기타 커밋되지 않은 모든 변경 사항



기본적으로 리뷰 패널은 **커밋되지 않은 변경 사항**에 초점을 맞춥니다. 범위를 다음으로 전환할 수도 있습니다.

  * **모든 브랜치 변경 사항** (기준 브랜치 대비 diff)
  * **마지막 턴의 변경 사항** (가장 최근 어시스턴트 턴만)



로컬에서 작업할 때는 **Unstaged** 변경 사항과 **Staged** 변경 사항 사이를 전환할 수도 있습니다.

## 리뷰 패널 탐색

  * 파일 이름을 클릭하면 일반적으로 선택한 에디터에서 해당 파일이 열립니다. 기본 에디터는 [설정](https://developers.openai.com/codex/app/settings)에서 선택할 수 있습니다.
  * 파일 이름 배경을 클릭하면 diff가 펼쳐지거나 접힙니다.
  * `Cmd`를 누른 상태에서 단일 줄을 클릭하면 선택한 에디터에서 해당 줄이 열립니다.
  * 어떤 변경 사항이 마음에 들면 [변경 사항을 스테이징하거나 원치 않는 변경 사항을 되돌릴 수 있습니다](https://developers.openai.com/codex/app/review#staging-and-reverting-files).



## 피드백을 위한 인라인 댓글

인라인 댓글을 사용하면 diff의 특정 줄에 피드백을 직접 연결할 수 있습니다. Codex를 올바른 수정 방향으로 안내하는 가장 빠른 방법인 경우가 많습니다.

인라인 댓글을 남기려면:

  1. 리뷰 패널을 엽니다.
  2. 댓글을 달고 싶은 줄에 마우스를 올립니다.
  3. 나타나는 **+** 버튼을 클릭합니다.
  4. 피드백을 작성하고 제출합니다.
  5. 모든 피드백을 마치면 스레드로 다시 메시지를 보냅니다.



댓글이 특정 줄에 고정되어 있기 때문에, Codex는 일반적인 지시보다 보통 더 정확하게 응답할 수 있습니다.

인라인 댓글은 리뷰 가이드로 처리됩니다. 댓글을 남긴 뒤에는 의도를 명확히 하는 후속 메시지를 보내세요. 예: “인라인 댓글을 반영하고 범위는 최소화해 주세요.”

## 코드 리뷰 결과

코드 리뷰를 실행할 때 `/review`를 사용하면, 댓글이 리뷰 패널에 직접 인라인으로 표시됩니다.

## 파일 스테이징 및 되돌리기

리뷰 패널에는 커밋 전에 diff를 정리할 수 있도록 Git 작업이 포함되어 있습니다.

여러 수준에서 변경 사항을 스테이징, 언스테이징 또는 되돌릴 수 있습니다.

  * **전체 diff** : 리뷰 헤더의 작업 버튼 사용 (예: “Stage all” 또는 “Revert all”)
  * **파일 단위** : 개별 파일을 스테이징, 언스테이징 또는 되돌리기
  * **hunk 단위** : 단일 hunk를 스테이징, 언스테이징 또는 되돌리기



작업의 일부만 수용하려면 스테이징을 사용하고, 버리려면 되돌리기를 사용하세요.

### 부분 스테이징 상태

Git은 같은 파일에서 스테이징된 변경 사항과 언스테이징된 변경 사항을 동시에 표현할 수 있습니다. 이 경우 패널에서 “같은 파일이 두 번” 보이는 것처럼 보일 수 있습니다. 이는 정상적인 Git 동작입니다.
