# Codex 웹

Codex는 코드를 읽고, 편집하고, 실행할 수 있는 OpenAI의 코딩 에이전트입니다. 더 빠르게 빌드하고, 버그를 수정하며, 익숙하지 않은 코드를 이해하는 데 도움을 줍니다. Codex 클라우드를 사용하면 Codex가 자체 클라우드 환경에서 (병렬 포함) 백그라운드 작업을 수행할 수 있습니다.

## Codex 웹 설정

[Codex](https://chatgpt.com/codex)에 접속하여 GitHub 계정을 연결하세요. 이렇게 하면 Codex가 리포지토리의 코드와 함께 작업하고 작업 결과로 풀 리퀘스트를 생성할 수 있습니다.

Plus, Pro, Business, Edu 또는 Enterprise 요금제에는 Codex가 포함됩니다. [포함 항목](https://developers.openai.com/codex/pricing)에 대해 자세히 알아보세요. 일부 Enterprise 워크스페이스는 Codex에 액세스하기 전에 [관리자 설정](https://developers.openai.com/codex/enterprise/admin-setup)이 필요할 수 있습니다.

## Codex 웹으로 작업하기

- [프롬프트 작성 알아보기](https://developers.openai.com/codex/prompting#prompts)

더 나은 결과를 얻으려면 프롬프트를 명확하게 쓰고, 제약 조건을 추가하며, 적절한 세부 수준을 선택하세요.

- [일반적인 워크플로](https://developers.openai.com/codex/workflows)

작업 위임, 변경사항 검토, 결과를 PR로 전환하는 입증된 패턴부터 시작하세요.

- [환경 구성](https://developers.openai.com/codex/cloud/environments)

클라우드에서 작업을 실행할 때 Codex가 사용할 리포지토리, 설정 단계, 도구를 선택하세요.

- [IDE 확장 기능에서 작업 위임](https://developers.openai.com/codex/ide/features#cloud-delegation)

편집기에서 클라우드 작업을 시작한 후 진행 상황을 모니터링하고 생성된 diff를 로컬에 적용하세요.

- [GitHub에서 위임하기](https://developers.openai.com/codex/integrations/github)

이슈와 풀 리퀘스트에 `@codex`를 태그하여 작업을 생성하고 GitHub에서 바로 변경을 제안하세요.

- [인터넷 액세스 제어](https://developers.openai.com/codex/cloud/internet-access)

클라우드 환경에서 Codex가 공개 인터넷에 접속할 수 있는지, 언제 활성화할지를 결정하세요.
