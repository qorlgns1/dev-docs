# 프롬프트 작성

Source URL: https://developers.openai.com/codex/prompting

## 프롬프트

프롬프트(사용자 메시지)를 보내 원하는 작업을 설명하면 Codex와 상호작용할 수 있습니다.

예시 프롬프트:
[code] 
    Explain how the transform module works and how other modules use it.
[/code]
[code] 
    Add a new command-line option `--json` that outputs JSON.
[/code]

프롬프트를 제출하면 Codex는 루프를 실행합니다. 모델을 호출한 뒤 모델 출력에 따라 파일 읽기, 파일 편집, 도구 호출 등 필요한 작업을 수행합니다. 작업이 완료되거나 사용자가 취소하면 이 과정이 끝납니다.

ChatGPT와 마찬가지로 Codex의 성능은 제공하는 지침의 질에 좌우됩니다. Codex에 프롬프트를 작성할 때 도움이 되는 팁은 다음과 같습니다.

  * Codex가 작업을 검증할 수 있을 때 더 높은 품질의 결과를 만듭니다. 문제 재현 단계, 기능 검증, 린트 및 pre-commit 검사 수행 방법을 포함하세요.
  * 작업을 더 작고 집중된 단계로 나누면 복잡한 작업을 Codex가 더 잘 처리합니다. 작은 작업일수록 Codex가 테스트하기 쉽고 사용자가 검토하기 쉽습니다. 작업 분할 방법이 확실하지 않다면 Codex에게 계획을 제안하도록 요청하세요.



Codex 프롬프트 작성에 대한 더 많은 아이디어는 [workflows](https://developers.openai.com/codex/workflows)를 참고하세요.

## 스레드

스레드는 단일 세션으로, 사용자의 프롬프트와 그에 따라 이어지는 모델 출력 및 도구 호출을 포함합니다. 하나의 스레드에는 여러 프롬프트를 넣을 수 있습니다. 예를 들어 첫 번째 프롬프트에서 Codex에게 기능 구현을 요청하고, 후속 프롬프트에서 테스트 추가를 요청할 수 있습니다.

Codex가 활발히 작업 중일 때 스레드가 “실행 중”이라고 합니다. 여러 스레드를 동시에 실행할 수 있지만, 동일한 파일을 두 스레드가 수정하지 않도록 주의하세요. 다른 프롬프트를 이어서 제공해 스레드를 나중에 다시 이어갈 수도 있습니다.

스레드는 로컬 또는 클라우드에서 실행할 수 있습니다.

  * **로컬 스레드**는 사용자의 머신에서 실행됩니다. Codex가 파일을 읽고 편집하며 명령을 실행할 수 있으므로 변경 사항을 확인하고 기존 도구를 그대로 사용할 수 있습니다. 워크스페이스 밖의 원치 않는 변경 위험을 줄이기 위해 로컬 스레드는 [sandbox](https://developers.openai.com/codex/security)에서 실행됩니다.
  * **클라우드 스레드**는 격리된 [environment](https://developers.openai.com/codex/cloud/environments)에서 실행됩니다. Codex가 저장소를 클론하고 작업 중인 브랜치를 체크아웃합니다. 병렬 작업이나 다른 기기에서 작업을 위임하고 싶을 때 유용합니다. 저장소로 클라우드 스레드를 사용하려면 먼저 코드를 GitHub에 푸시하세요. 현재 작업 상태를 포함해 [로컬 머신에서 작업을 위임](https://developers.openai.com/codex/ide/cloud-tasks)할 수도 있습니다.



## 컨텍스트

프롬프트를 제출할 때 Codex가 사용할 수 있도록 관련 파일과 이미지 같은 컨텍스트를 포함하세요. Codex IDE 확장은 열려 있는 파일 목록과 선택한 텍스트 범위를 자동으로 컨텍스트에 추가합니다.

에이전트가 작업하면서 파일 내용, 도구 출력, 수행 기록 및 남은 과업에서 컨텍스트를 수집합니다.

스레드의 모든 정보는 모델의 **context window** 안에 들어가야 하며, 이 크기는 모델마다 다릅니다. Codex는 남은 공간을 모니터링하고 보고합니다. 작업이 길어지면 Codex가 자동으로 관련 정보를 요약하고 덜 관련된 세부 정보를 버리는 방식으로 컨텍스트를 **compact**할 수 있습니다. 반복적으로 압축해 Codex는 여러 단계에 걸쳐 복잡한 작업을 계속 진행할 수 있습니다.
