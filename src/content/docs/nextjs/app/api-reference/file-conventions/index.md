# 파일 시스템 규칙

2026년 2월 20일에 마지막으로 업데이트됨

- [default.js](https://nextjs.org/docs/app/api-reference/file-conventions/default)
  - default.js 파일에 대한 API 레퍼런스입니다.
- [동적 세그먼트](https://nextjs.org/docs/app/api-reference/file-conventions/dynamic-routes)
  - Dynamic Segments동적 라우트 세그먼트는 동적 데이터를 기반으로 라우트 세그먼트를 프로그래밍 방식으로 생성하는 데 사용할 수 있습니다.
- [error.js](https://nextjs.org/docs/app/api-reference/file-conventions/error)
  - error.js 특별 파일에 대한 API 레퍼런스입니다.
- [forbidden.js](https://nextjs.org/docs/app/api-reference/file-conventions/forbidden)
  - forbidden.js 특별 파일에 대한 API 레퍼런스입니다.
- [instrumentation.js](https://nextjs.org/docs/app/api-reference/file-conventions/instrumentation)
  - instrumentation.js 파일에 대한 API 레퍼런스입니다.
- [instrumentation-client.js](https://nextjs.org/docs/app/api-reference/file-conventions/instrumentation-client)
  - Next.js 애플리케이션의 프런트엔드 성능을 추적하고 모니터링하기 위한 클라이언트 측 계측을 추가하는 방법을 배웁니다.
- [인터셉팅 라우트](https://nextjs.org/docs/app/api-reference/file-conventions/intercepting-routes)
  - Intercepting Routes브라우저 URL을 가리면서 현재 레이아웃 내에 새 라우트를 로드해 모달과 같은 고급 라우팅 패턴에 유용한 인터셉트 라우트를 사용하세요.
- [layout.js](https://nextjs.org/docs/app/api-reference/file-conventions/layout)
  - layout.js 파일에 대한 API 레퍼런스입니다.
- [loading.js](https://nextjs.org/docs/app/api-reference/file-conventions/loading)
  - loading.js 파일에 대한 API 레퍼런스입니다.
- [mdx-components.js](https://nextjs.org/docs/app/api-reference/file-conventions/mdx-components)
  - mdx-components.js 파일에 대한 API 레퍼런스입니다.
- [not-found.js](https://nextjs.org/docs/app/api-reference/file-conventions/not-found)
  - not-found.js 파일에 대한 API 레퍼런스입니다.
- [page.js](https://nextjs.org/docs/app/api-reference/file-conventions/page)
  - page.js 파일에 대한 API 레퍼런스입니다.
- [병렬 라우트](https://nextjs.org/docs/app/api-reference/file-conventions/parallel-routes)
  - Parallel Routes동시에 하나 이상의 페이지를 동일한 뷰에 렌더링하고 각각 독립적으로 탐색할 수 있는 고도로 동적인 애플리케이션 패턴입니다.
- [proxy.js](https://nextjs.org/docs/app/api-reference/file-conventions/proxy)
  - proxy.js 파일에 대한 API 레퍼런스입니다.
- [public](https://nextjs.org/docs/app/api-reference/file-conventions/public-folder)
  - Next.js는 public 디렉터리에서 이미지와 같은 정적 파일을 제공할 수 있도록 하며, 작동 방식을 여기에서 배울 수 있습니다.
- [route.js](https://nextjs.org/docs/app/api-reference/file-conventions/route)
  - route.js 특별 파일에 대한 API 레퍼런스입니다.
- [라우트 그룹](https://nextjs.org/docs/app/api-reference/file-conventions/route-groups)
  - Route GroupsRoute Groups를 사용해 Next.js 애플리케이션을 서로 다른 섹션으로 구분할 수 있습니다.
- [라우트 세그먼트 구성](https://nextjs.org/docs/app/api-reference/file-conventions/route-segment-config)
  - Route Segment ConfigNext.js 라우트 세그먼트 옵션을 구성하는 방법을 알아보세요.
- [src](https://nextjs.org/docs/app/api-reference/file-conventions/src-folder)
  - 루트 `pages` 디렉터리 대신 `src` 폴더 아래에 페이지를 저장하세요.
- [template.js](https://nextjs.org/docs/app/api-reference/file-conventions/template)
  - template.js 파일에 대한 API 레퍼런스입니다.
- [unauthorized.js](https://nextjs.org/docs/app/api-reference/file-conventions/unauthorized)
  - unauthorized.js 특별 파일에 대한 API 레퍼런스입니다.
- [개요](https://nextjs.org/docs/app/api-reference/file-conventions/metadata)
  - Metadata Files메타데이터 파일 규칙에 대한 API 문서입니다.
