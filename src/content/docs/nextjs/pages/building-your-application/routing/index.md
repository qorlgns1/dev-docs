# 애플리케이션 빌드: 라우팅 | Next.js

원본 URL: https://nextjs.org/docs/pages/building-your-application/routing

# 라우팅

마지막 업데이트 2026년 2월 20일

페이지 라우터는 페이지 개념을 기반으로 한 파일 시스템 기반 라우터를 제공합니다. `pages` 디렉터리에 파일을 추가하면 자동으로 라우트로 사용할 수 있습니다. 페이지 라우터의 라우팅에 대해 더 알아보세요:

- [페이지와 레이아웃](https://nextjs.org/docs/pages/building-your-application/routing/pages-and-layouts)
  - 페이지 및 레이아웃페이지 라우터로 첫 페이지와 공유 레이아웃을 만들어 보세요.

- [동적 라우트](https://nextjs.org/docs/pages/building-your-application/routing/dynamic-routes)
  - 동적 라우트는 URL에 사용자 정의 매개변수를 추가할 수 있는 페이지입니다. 동적 라우트를 만들고 더 알아보세요.

- [링크 및 탐색](https://nextjs.org/docs/pages/building-your-application/routing/linking-and-navigating)
  - 링킹 및 네비게이션Next.js에서 내비게이션이 작동하는 방식과 Link 컴포넌트, `useRouter` 훅을 사용하는 방법을 알아보세요.

- [커스텀 App](https://nextjs.org/docs/pages/building-your-application/routing/custom-app)
  - 사용자 정의 AppNext.js가 사용하는 기본 App 컴포넌트를 재정의하여 페이지 초기화와 모든 페이지에 지속되는 레이아웃을 제어하세요.

- [커스텀 Document](https://nextjs.org/docs/pages/building-your-application/routing/custom-document)
  - 사용자 정의 DocumentNext.js가 추가하는 기본 문서 마크업을 확장하세요.

- [API 라우트](https://nextjs.org/docs/pages/building-your-application/routing/api-routes)
  - Next.js는 API 라우트를 지원하여 Next.js 앱을 벗어나지 않고 API를 구축할 수 있습니다. 작동 방식을 알아보세요.

- [커스텀 오류](https://nextjs.org/docs/pages/building-your-application/routing/custom-error)
  - 사용자 정의 오류기본 제공 Error 페이지를 재정의하고 확장하여 사용자 정의 오류를 처리하세요.
