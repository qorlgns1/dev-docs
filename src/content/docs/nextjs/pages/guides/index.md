# Pages Router: 가이드 | Next.js

출처 URL: https://nextjs.org/docs/pages/guides

# 가이드

마지막 업데이트 2026년 2월 20일

- [분석](https://nextjs.org/docs/pages/guides/analytics)
  - 애널리틱스Next.js로 페이지 성능을 측정하고 추적하세요.

- [인증](https://nextjs.org/docs/pages/guides/authentication)
  - Next.js에서 인증을 구현하는 방법을 배우며, 모범 사례, 라우트 보호, 권한 부여 기법, 세션 관리를 다룹니다.

- [Babel](https://nextjs.org/docs/pages/guides/babel)
  - Next.js가 추가한 Babel 프리셋을 사용자 구성으로 확장하세요.

- [CI 빌드 캐싱](https://nextjs.org/docs/pages/guides/ci-build-caching)
  - Next.js 빌드를 캐시하도록 CI를 구성하는 방법을 알아보세요.

- [콘텐츠 보안 정책](https://nextjs.org/docs/pages/guides/content-security-policy)
  - Next.js 애플리케이션에 CSP를 설정하는 방법을 배워보세요.

- [CSS-in-JS](https://nextjs.org/docs/pages/guides/css-in-js)
  - Next.js에서 CSS-in-JS 라이브러리를 사용하세요.

- [커스텀 서버](https://nextjs.org/docs/pages/guides/custom-server)
  - 커스텀 서버를 사용해 Next.js 앱을 프로그래밍 방식으로 시작하세요.

- [디버깅](https://nextjs.org/docs/pages/guides/debugging)
  - VS Code나 Chrome DevTools로 Next.js 애플리케이션을 디버깅하는 방법을 알아보세요.

- [드래프트 모드](https://nextjs.org/docs/pages/guides/draft-mode)
  - Next.js에는 정적 페이지와 동적 페이지를 전환하는 드래프트 모드가 있습니다. Pages Router와 함께 작동 방식을 배워보세요.

- [환경 변수](https://nextjs.org/docs/pages/guides/environment-variables)
  - Next.js 애플리케이션에서 환경 변수를 추가하고 접근하는 방법을 배워보세요.

- [폼](https://nextjs.org/docs/pages/guides/forms)
  - Next.js에서 폼 제출과 데이터 변경을 처리하는 방법을 알아보세요.

- [ISR](https://nextjs.org/docs/pages/guides/incremental-static-regeneration)
  - 인크리멘털 정적 재생성을 사용하여 런타임에 정적 페이지를 생성하거나 업데이트하는 방법을 알아보세요.

- [계측](https://nextjs.org/docs/pages/guides/instrumentation)
  - 인스트루멘테이션Next.js 앱에서 서버 시작 시 코드를 실행할 수 있도록 인스트루멘테이션을 사용하는 방법을 배우세요.

- [국제화](https://nextjs.org/docs/pages/guides/internationalization)
  - Next.js는 국제화된 라우팅과 언어 감지를 기본 지원합니다. 여기에서 자세히 알아보세요.

- [지연 로딩](https://nextjs.org/docs/pages/guides/lazy-loading)
  - 가져온 라이브러리와 React 컴포넌트를 지연 로딩하여 전체 로딩 성능을 개선하세요.

- [MDX](https://nextjs.org/docs/pages/guides/mdx)
  - 마크다운 파일에서 JSX를 작성할 수 있도록 MDX를 구성하는 방법을 배워보세요.

- [개요](https://nextjs.org/docs/pages/guides/migrating)
  - 마이그레이션인기 프레임워크에서 Next.js로 마이그레이션하는 방법을 알아보세요.

- [멀티 존](https://nextjs.org/docs/pages/guides/multi-zones)
  - Next.js Multi-Zones를 사용해 여러 Next.js 앱을 단일 도메인에 배포하는 마이크로 프런트엔드를 구축하세요.

- [OpenTelemetry](https://nextjs.org/docs/pages/guides/open-telemetry)
  - OpenTelemetry로 Next.js 앱을 계측하는 방법을 배워보세요.

- [패키지 번들링](https://nextjs.org/docs/pages/guides/package-bundling)
  - 애플리케이션의 서버 및 클라이언트 번들을 최적화하는 방법을 알아보세요.

- [PostCSS](https://nextjs.org/docs/pages/guides/post-css)
  - Next.js가 추가한 PostCSS 구성과 플러그인을 사용자 정의로 확장하세요.

- [미리보기 모드](https://nextjs.org/docs/pages/guides/preview-mode)
  - 프리뷰 모드Next.js에는 정적으로 생성된 페이지를 위한 프리뷰 모드가 있습니다. 여기에서 작동 방식을 알아보세요.

- [프로덕션](https://nextjs.org/docs/pages/guides/production-checklist)
  - Next.js 애플리케이션을 프로덕션에 배포하기 전에 최고의 성능과 사용자 경험을 보장하기 위한 권장 사항입니다.

- [리다이렉트](https://nextjs.org/docs/pages/guides/redirecting)
  - 리디렉션Next.js에서 리디렉션을 처리하는 다양한 방법을 알아보세요.

- [Sass](https://nextjs.org/docs/pages/guides/sass)
  - Next.js 애플리케이션에서 Sass를 사용하는 방법을 배워보세요.

- [스크립트](https://nextjs.org/docs/pages/guides/scripts)
  - 내장 Script 컴포넌트로 서드파티 스크립트를 최적화하세요.

- [셀프 호스팅](https://nextjs.org/docs/pages/guides/self-hosting)
  - Node.js 서버, Docker 이미지 또는 정적 HTML 파일(정적 내보내기)에서 Next.js 애플리케이션을 직접 호스팅하는 방법을 알아보세요.

- [정적 내보내기](https://nextjs.org/docs/pages/guides/static-exports)
  - Next.js는 정적 사이트나 SPA로 시작한 뒤 나중에 서버가 필요한 기능을 선택적으로 사용할 수 있습니다.

- [Tailwind CSS](https://nextjs.org/docs/pages/guides/tailwind-v3-css)
  - Tailwind CSS로 Next.js 애플리케이션을 스타일링하세요.

- [개요](https://nextjs.org/docs/pages/guides/testing)
  - 테스팅Next.js를 Cypress, Playwright, Vitest, Jest 등 세 가지 일반적인 테스트 도구와 설정하는 방법을 알아보세요.

- [서드파티 라이브러리](https://nextjs.org/docs/pages/guides/third-party-libraries)
  - `@next/third-parties` 패키지로 애플리케이션의 서드파티 라이브러리 성능을 최적화하세요.

- [개요](https://nextjs.org/docs/pages/guides/upgrading)
  - 업그레이드최신 Next.js 버전으로 업그레이드하는 방법을 알아보세요.
