# 확장 구성 | Sentry for Next.js

*
- [API](https://docs.sentry.io/platforms/javascript/guides/nextjs/configuration/apis.md)

  SDK의 API에 대해 자세히 알아보세요.

*
- [옵션](https://docs.sentry.io/platforms/javascript/guides/nextjs/configuration/options.md)

  옵션을 통해 SDK를 구성하는 방법을 자세히 알아보세요. 이러한 옵션은 init 함수에 전달되므로 SDK가 처음 초기화될 때 설정됩니다.

*
- [빌드 옵션](https://docs.sentry.io/platforms/javascript/guides/nextjs/configuration/build.md)

  앱의 빌드 프로세스를 제어하기 위해 Next.js 구성에서 사용하는 설정 옵션을 알아보세요.

*
- [통합](https://docs.sentry.io/platforms/javascript/guides/nextjs/configuration/integrations.md)

  통합이 일반적인 라이브러리와 환경을 자동으로 지원하여 SDK 기능을 확장하는 방법을 자세히 알아보세요.

*
- [전송](https://docs.sentry.io/platforms/javascript/guides/nextjs/configuration/transports.md)

  전송(Transport)을 사용하면 이벤트를 Sentry에 전달하는 방식을 변경할 수 있습니다.

*
- [릴리스 및 상태](https://docs.sentry.io/platforms/javascript/guides/nextjs/configuration/releases.md)

  SDK를 구성해 릴리스 정보를 Sentry에 알리는 방법을 알아보세요.

*
- [필터링](https://docs.sentry.io/platforms/javascript/guides/nextjs/configuration/filtering.md)

  Sentry에 보고되는 이벤트에 필터링을 설정하도록 JavaScript SDK를 구성하는 방법을 자세히 알아보세요.

*
- [이벤트 루프 블로킹 감지](https://docs.sentry.io/platforms/javascript/guides/nextjs/configuration/event-loop-block.md)

  Node.js 애플리케이션에서 블로킹된 이벤트 루프를 모니터링합니다.

*
- [트리 셰이킹](https://docs.sentry.io/platforms/javascript/guides/nextjs/configuration/tree-shaking.md)

  사용하지 않는 코드를 트리 셰이킹하여 Sentry 번들 크기를 줄이는 방법을 알아보세요.
