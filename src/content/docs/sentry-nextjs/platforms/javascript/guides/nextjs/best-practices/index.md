# 특수 사용 사례 | Next.js용 Sentry

*
- [Cloudflare Workers에 배포하기](https://docs.sentry.io/platforms/javascript/guides/nextjs/best-practices/deploying-on-cloudflare.md)

  Cloudflare Workers에 배포된 Next.js 애플리케이션에 맞게 Sentry를 구성하는 방법을 알아보세요.

*
- [공유 환경 / 브라우저 확장 프로그램](https://docs.sentry.io/platforms/javascript/guides/nextjs/best-practices/shared-environments.md)

  공유 환경(예: 브라우저 확장 프로그램 또는 VSCode 확장 프로그램)에서 Sentry를 사용하는 방법을 알아보세요.

*
- [마이크로 프론트엔드](https://docs.sentry.io/platforms/javascript/guides/nextjs/best-practices/micro-frontends.md)

  마이크로 프론트엔드 또는 module federation을 사용할 때 오류의 출처를 식별하고 이벤트를 서로 다른 Sentry 프로젝트로 라우팅하는 방법을 알아보세요.

*
- [여러 Sentry 인스턴스](https://docs.sentry.io/platforms/javascript/guides/nextjs/best-practices/multiple-sentry-instances.md)

  자체 클라이언트를 생성해 여러 Sentry 인스턴스를 관리하는 방법을 알아보세요.

*
- [오프라인 캐싱](https://docs.sentry.io/platforms/javascript/guides/nextjs/best-practices/offline-caching.md)

  오프라인 상태에서 Sentry 이벤트를 캐시하는 방법을 알아보세요.

*
- [Web Workers](https://docs.sentry.io/platforms/javascript/guides/nextjs/best-practices/web-workers.md)

  Web Workers API에서 Sentry의 Browser SDK를 사용하는 방법을 알아보세요.

*
- [Sentry Testkit](https://docs.sentry.io/platforms/javascript/guides/nextjs/best-practices/sentry-testkit.md)

  올바른 흐름 추적 또는 오류가 Sentry로 전송되는지 검증하되, 실제로 Sentry 서버로 전송하지 않는 방법을 알아보세요.
